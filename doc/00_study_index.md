# 机器人系统学习总索引

这套仓库的学习难点不在单个模块，而在“低时延数据链路 + 机器人执行链路 + 端侧 Agent + 工业化部署”是一起工作的。建议不要按目录乱看，而是按下面顺序推进。

## 1. 推荐学习顺序

1. 先看 `01_ros_shm_low_latency.md`。这里解释仓库最核心的设计思想：大数据走共享内存，ROS 话题只传句柄。
2. 再看 `02_ros1_ros2_switching_latency.md`。这一步帮助你理解作者为什么同时保留 ROS1 和 ROS2 两套低时延实现，以及怎么做时延对比。
3. 然后看 `04_vision_pipeline.md` 和 `08_lidar_pipeline.md`。这是感知主链路，决定后面的跟踪、导航、Agent 控制是否成立。
4. 接着看 `05_upper_computer_control.md` 和 `06_stm32_mcu_firmware.md`。这两份文档把 `/cmd_vel` 如何真正变成电机 PWM、编码器反馈和里程计说清楚。
5. 再看 `07_navigation_system.md` 与 `11_urdf_model.md`。这里会把定位、路径规划、TF 树和传感器安装位姿串起来。
6. 最后看 `03_edge_llm_agent_online_offline.md`、`09_edge_monitoring.md`、`10_config_and_deployment.md`。这部分解释端侧 Agent、监控和工业化交付。

## 2. 每份专题文档怎么用

每份文档都按同一模板组织：

- 这个功能到底解决什么问题
- 先看哪些文件
- 启动路径 / 数据流 / 控制流
- 亮点实现细节
- 调试与验证方法
- 学习路线（30 分钟 / 2 小时 / 1 天）
- 学完后应该能回答什么问题

这样做的目的不是“罗列文件”，而是帮你在短时间内建立三层认知：

- 第一层：这个模块在整机里承担什么职责。
- 第二层：消息、线程、进程、设备之间如何流动。
- 第三层：作者真正做了哪些低时延、稳定性、工程化优化。

## 3. 建议的实操节奏

- ROS1 主工程先执行 `catkin_make`，然后 `source devel/setup.bash`。
- ROS2 工作区执行 `cd ros2_ws && colcon build --symlink-install`，再 `source install/setup.bash`。
- 如果你只有半天，优先看 01、04、05、03 四份。
- 如果你要做复现或二次开发，11 份都要看，尤其是 02、08、10，这三份决定迁移成本和部署稳定性。

## 4. 需要提前知道的边界

- README 提到“大模型微调、量化”，但仓库里最完整的是在线/离线推理部署与指令执行链路，不是完整训练平台。对应文档会把“仓库真实实现”和“README 宣称能力”分开写。
- README 提到 ROS1/ROS2 时延性能分析，但仓库主要提供对照实现和测量方法，不是现成的最终 benchmark 报告。对应文档会告诉你应该如何自己复现测量。
- 很多模块同时兼容 RK 平台与 x86 调试路径。理解“硬件加速分支”和“桌面回退分支”的差异，是读懂代码的关键。
